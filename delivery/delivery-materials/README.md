# 交付材料整理

## 这个 Skill 是干什么的

`delivery-materials` 用于把软件项目的功能范围、提交记录、接口变更、页面操作、截图和验收要求，整理成正式、业主可读的中文交付材料。

## 适合什么时候用

- 需要快速生成一套项目交付材料。
- 需要把 Git 提交或功能清单转成验收口径。
- 需要编写用户使用手册，包含页面操作、接口调用说明和截图占位。
- 需要编写功能测试、性能测试、测试验收报告或运维手册。
- 需要把 Markdown 交付材料转换为 Word 文档。
- 截图已准备好，需要按占位符嵌入 Word。

## 默认输出

| 文件 | 内容 |
| --- | --- |
| `00-交付材料范围说明.md` | 交付范围、不纳入范围、材料清单和证据来源。 |
| `01-用户使用手册.md` | 用户角色、菜单路径、操作步骤、接口调用和截图占位。 |
| `02-功能测试.md` | 黑盒测试和白盒测试设计。 |
| `03-性能测试.md` | 性能场景、指标口径、代码侧性能检查和结论。 |
| `04-测试验收报告.md` | 验收范围、依据、验证情况、风险和验收结论。 |
| `05-运维手册.md` | 环境、部署、配置、监控、日志、备份、回滚和故障处理。 |
